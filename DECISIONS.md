## D-001 (2026-07-25, Day 1) — 프로젝트 주제로 운영 티켓 시스템 선정

### 결정

부트캠프 추천 5개 중 **운영 티켓 / 작업 관리 시스템** 채택.

### 검토한 대안

| 후보 | 검토 결과 |
| --- | --- |
| 1. 운영 티켓 / 작업 관리 (선택) | 6 필수 기능 자연스러운 매핑, 도메인 학습 비용 낮음, 비전공자도 1분 이해 |
| 2. 커머스 주문/클레임 | 트랜잭션 깊이는 좋지만 도메인 학습 비용 큼, 결제 로직이 들어가면 보안 부담 |
| 3. 채용/과제/면접 파이프라인 | 부트캠프 정체성과 직결되지만 데이터 모델이 단순해 트랜잭션 학습 효과 약함 |
| 4. 콘텐츠 검수/승인 | UI 비중 커져 백엔드 학습 시간 분산 |
| 5. 의료/상담/예약 | 도메인 규제 (개인정보) 부담 |

### 선택 근거

- 6 공통 필수 기능 모두가 도메인적으로 자연스럽게 등장 (권한·트랜잭션·검색·캐시·비동기·AI)
- 신청자/운영자 2 페르소나로 권한 분기 자연스럽게 시연 가능
- AI 보조 기능 후보가 풍부 (요약, 우선순위 추천, 답변 초안)
- 다음 기수에서 다른 주제로 비교 학습하기 쉬운 보편적 도메인

### 책임자

팀 4명 합의 (Day 1 오전 미팅)

## D-002 (2026-07-25, Day 1) — 라이프사이클 단계별 책임자 분담

### 결정

| 단계 | 책임자 | 기술 영역 |
| --- | --- | --- |
| 기획 | 팀원 A | 인증/권한 |
| 코딩 | 팀 전원 | (각자 본인 영역) |
| 테스트 | 팀원 B | 핵심 조회 |
| 리뷰 | 팀원 C | 동시성/상태 전이 |
| 배포·운영 | 팀원 D | 캐시/비동기/AI |

### 검토한 대안

- A. 라이프사이클 1 단계 = 1명 풀 책임 (선택)
- B. 모든 단계를 팀 전원이 똑같이 분담
- C. 라이프사이클 무시, 기술 영역만으로 분담

### 선택 근거

- A는 단계별 deep ownership이 생겨 책임 명확
- B는 누구도 진짜 책임지지 않는 위험
- C는 Week 8에서 익힌 라이프사이클 학습이 사라짐
- A + 팀 전원이 코딩 단계는 공동 (본인 기술 영역)

### 책임자

팀 4명 합의 (Day 1 오후 미팅)

## D-004 (2026-07-25, Day 1) — AI 보조 기능 범위 선정

### 결정

3개 기능 모두 구현하되 우선순위:

1. **티켓 요약** (필수) — 본문 → 1줄 요약, 운영자 인계 시 사용
2. **우선순위 추천** (필수) — LOW/MEDIUM/HIGH/URGENT 후보 제시, 운영자 채택률 측정
3. **댓글 답변 초안** (선택, 시간되면) — 비슷한 과거 티켓 참고로 답변 초안

### 검토한 대안

- A. 1개만 깊게
- B. 3개 모두 (선택)
- C. 5개 (요약·추천·답변·분류·유사 티켓 검색)

### 선택 근거

- 시연용으로 1개는 약함, 5개는 깊이 부족
- 3개가 PRD의 페르소나 페인포인트(트리아지 시간 + 인계 + 답변)와 자연스럽게 매핑
- 검증 가능한 정확도 지표 (운영자 채택률) 가짐

### 책임자

팀원 D 주도

## D-005 (2026-07-25, Day 1) — 라이프사이클 도구 선택

### 결정

| 단계 | 도구 | 책 챕터 |
| --- | --- | --- |
| 기획 | Jira MCP + Claude Code | 9주차 03 |
| 코딩 | claude.md / Commands / Hooks / gh CLI | 7주차 04~07, 9주차 02 |
| 테스트 | Playwright MCP | 9주차 06 |
| 리뷰 | Claude GitHub Actions (`anthropics/claude-code-action@v1`) | 10주차 01 |
| 배포·운영 | Sentry MCP + Docker Compose | 10주차 06 |

### 검토한 대안 (단계별)

- 리뷰: Claude Actions vs CodeRabbit
  - Claude Actions: 우리 팀 코딩 규칙(`CLAUDE.md`)을 prompt로 직접 전달 가능 (선택)
  - CodeRabbit: 더 풍부한 기능, 단 우리 팀 룰 학습 시간 필요
- 배포: Docker Compose vs Terraform AWS
  - Docker Compose: 시연용 단순 (선택)
  - Terraform AWS: 운영급이지만 Week 10 분량 초과

### 선택 근거

- Week 9 한정 시연이라 도구 깊이보다 5 단계 모두 적용에 무게
- Claude Actions가 우리 팀 헌법을 가장 자연스럽게 강제
